00 · OVERVIEW
Project Brief
WHAT WE ARE BUILDING AND WHY IT MATTERS

📷 AR Filter System

Snapchat-quality lenses built from scratch using custom GLSL fragment shaders, CameraX, ARCore, and MediaPipe Face Landmarker. No third-party lens SDK — full design control, no licensing fees, no child-data leakage.

🐾 Pet System

A modernised Tamagotchi where captured monsters become emotional companions. Inspired by the "If" film. Soft, expressive, never punishing — no death mechanic.

01 · PHILOSOPHY
Design Philosophy
PRINCIPLES EVERY SCREEN MUST HONOUR

WHIMSICAL · NOT CHILDISH

Playful with craft. Surprising details. Every interaction has personality.

DARK-SAFE FIRST

Kids hunt monsters at night. All UI readable at max darkness. OLED-friendly.

NO DEATH, NO PUNISHMENT

Monsters are immortal friends. Low stats = sad moods, not loss.

KIDS-FIRST ACCESSIBILITY

48×48dp minimum. No spoiler screen-reading. Haptic feedback everywhere.

EMOTIONAL EXPRESSIVENESS

Bond is the reward. Every interaction confirms the monster felt it.

PERFORMANCE IS DESIGN

≤16ms/frame on Tier B. Fallbacks first-class, not afterthoughts.

02.2 · VISUAL SYSTEM
Motion & Haptics
ANIMATIONS · HAPTIC PATTERNS · LOTTIE INTEGRATION
🎵
LOTTIE RULES
All idle animations use iterations = LottieConstants.IterateForever. Triggered animations play once then return to idle via isPlaying state tracking. Bundle all Lottie JSON in assets/pet/moods/ — no remote loading.
03 · AR FILTER SYSTEM
Filter Architecture
COMPOSABLE GLSL PIPELINE · ARCORE · CAMERAX · MEDIAPIPE
⚙️
PIPELINE FLOW
CameraX → ARCore → MediaPipe FaceLandmarker → UniformBus → FilterLayerManager → OpenGL ES 3.0 FBO chain → Display

FilterLayerManager

Maintains active shader stack as ordered List<FilterLayer>. Stack ordered cheapest → most expensive for Tier B fallback.

UniformBus

StateFlow-backed data channel feeding real-time values into GLSL uniforms. Never query MediaPipe from GL thread.
